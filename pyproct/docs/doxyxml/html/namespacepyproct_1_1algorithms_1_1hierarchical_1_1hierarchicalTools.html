def pyproct.algorithms.hierarchical.hierarchicalTools.find_cutoff_limit (   starting_cutoff,
  min_clusters,
  max_clusters,
  grain,
  hie_algorithm 
)

Created on 21/05/2012.

Author:
: victor

Calculates one of the limits of the cutoff range by adding 'grain' to the starting cutoff until the number of clusters of the produced clustering is into the expected range. The direction can be controlled with the sign of 'grain'.

Parameters:
starting_cutoff,:A guess of the smaller cutoff we can use. It can be 0
min_clusters,:The minimum number of clusters allowed by clustering.
max_clusters,:The maximum number of clusters allowed by clustering.
grain,:The cutoff step that will be used to discover the cutoff range. If it is positive it is going to move to the right, if negative to the left. Exiting conditions are also based on this.
hie_algorithm,:The hierarchical algorithm instance (holds the hie matrix, so that it has not to be calculated each time we get a clustering).

Searches for the range of cutoffs where we are going to get a number of clusters inside the allowed number of clusters [min_clusters,max_clusters].

Returns a dictionary indexed by number of clusters where each item is a tuple of (cutoff,Clustering). A dictionary is used because all clusterings with same number of clusters would be the same (and in this way we get the set without repetition).

Parameters:
cutoff_range_begin,:A guess of the smaller cutoff we can use. It can be 0
cutoff_range_end,:A guess of the bigger cutoff we can use.
refine_grain,:Positive number that defines the cutoff step that will be used to discover the cutoff range.
hie_algorithm,:The hierarchical algorithm instance (holds the hie matrix, so that it has not to be calculated each time we get a clustering).
min_clusters,:The minimum number of clusters allowed by clustering.
max_clusters,:The maximum number of clusters allowed by clustering.

def pyproct.algorithms.hierarchical.hierarchicalTools.get_cutoff_range (   starting_cutoff,
  ending_cutoff,
  min_clusters,

)

Returns the cutoff range where the cutoff is inside the range defined by [min_clusters, max_clusters].

It first calculates the left limit (from the minimum value of the cutoff, and the smaller number of clusters) and then the right limit (from the maximum value of the cutoff).

Parameters:
starting_cutoff,:A guess of the smaller cutoff we can use. It can be 0
ending_cutoff,:A guess of the bigger cutoff we can use.
min_clusters,:The minimum number of clusters allowed by clustering.
max_clusters,:The maximum number of clusters allowed by clustering.
grain,:Positive number that defines the cutoff step that will be used to discover the cutoff range.
hie_algorithm,:The hierarchical algorithm instance (holds the hie matrix, so that it has not to be calculated each time we get a clustering).
