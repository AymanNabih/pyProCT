  keep_header = False,
  write_frame_number_instead_of_correlative_model_number = False 
)

It extracts some frames from one trajectory and writes them down in an opened file handler.

Parameters:
file_in_handler,:Input file handler.
number_of_frames,:Number of models in the input trajectory.
file_out_handler,:File handler to write the model to.
frames_to_save,:An array with the frame numbers to copy in the output trajectory. For example: [0,1,2] will store the three first frames of the input trajectory into the second.
INITIAL_TAG,:Is the tag preceding the atom pdb data. Usually is "MODEL".
END_TAG,:Is the last tag in a model, after the pdb atom data. Usually is "ENDMDL" or "TER".
keep_header,:Will try to keep any header previous to INITIAL_TAG, such as "REMARK" lines.
use_frame_number_as_model,:If true, it will use the number of frame in 'frames_to_save' as model number in the new file. If not, models of the new file will have sequential numbering.

def pyproct.tools.pdbTools.write_a_tfile_model_into_other_tfile (   file_in_handler,
  file_out_handler,
  current_model,
  INITIAL_TAG,
  END_TAG,
  skip = False,
  keep_header = False 
)

It writes the next model found in a trajectory unless the skip parameter is set to 'True'.

It makes the input file handler reading cursor to be placed the line after the end of the model.

Parameters:
file_in_handler,:Input file handler.
file_out_handler,:File handler to write the model to.
current_model,:Is the model number we will write in the output handler.
INITIAL_TAG,:Is the tag preceding the atom pdb data. Usually is "MODEL".
END_TAG,:Is the last tag in a model, after the pdb atom data. Usually is "ENDMDL" or "TER".
skip,:If true, the model will not be written to the output file, but the cursor of the input_file will be moved anyway.
keep_header,:Will try to keep any header previous to INITIAL_TAG, such as "REMARK" lines.
